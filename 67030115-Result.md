## 🔍 คำถามทบทวน พร้อมคำตอบ

### 1. Docker vs Native Setup
**ข้อดีของ Docker:**
- มี environment ที่ consistent, ลดปัญหา dependency conflicts  
- ติดตั้งครั้งเดียว ใช้ได้ทุก OS ที่รองรับ Docker  
- ง่ายต่อการแชร์และ reproduce project  
- แยกสภาพแวดล้อมไม่กระทบ host system  

**ข้อดีของ Native Setup:**
- เข้าถึง hardware ได้โดยตรง  
- ไม่มี overhead ของ container  
- การ debug บางอย่างง่ายกว่า  

---

### 2. Build Process (ESP-IDF in Docker)
1. เขียน source code บน host  
2. mount project folder เข้า container  
3. Docker ใช้ ESP-IDF toolchain ภายใน container  
4. CMake และ Ninja ประมวลผลไฟล์ source -> object files  
5. linker สร้างไฟล์ binary (.elf, .bin)  
6. binary สามารถ flash ลง ESP32 ผ่าน USB  

---

### 3. CMake Files
- **Top-level CMakeLists.txt:** กำหนด project, include subdirectories, และ options  
- **Component-level CMakeLists.txt:** กำหนด source files, dependencies, และ export targets  
- **ใน Docker environment:** CMake ใช้ toolchain ภายใน container ทำให้ build ได้เหมือน native environment แต่ไม่กระทบ host  

---

### 4. Git Ignore
- ป้องกันไม่ให้ไฟล์ build, temporary, หรือ config ของ container ถูก commit  
- ตัวอย่างไฟล์ที่ควร ignore: `build/`, `.vscode/`, `.espressif/`, `*.bin`, `*.elf`  
- ทำให้ repo สะอาดและลดขนาด repository  

---

### 5. Container Persistence
- **ข้อมูลหายเมื่อ restart container:**  
  - การติดตั้ง package ภายใน container (ถ้าไม่ได้ใช้ volume)  
  - temporary files ใน filesystem ของ container  
- **ข้อมูลอยู่ต่อ:**  
  - ถ้า mount host folder เช่น project folder หรือ ESP-IDF configs  
  - volume ที่กำหนดไว้ เช่นสำหรับ toolchain cache  

---

### 6. Development Workflow
| Aspect | Docker | Native System |
|--------|--------|---------------|
| Environment Setup | ง่าย, reproducible | ต้องติดตั้งเองทุกเครื่อง |
| Build & Flash | ผ่าน container, isolated | ใช้ toolchain บน host |
